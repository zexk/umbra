SCHEME   dark, purple
SCOPE    terminal · gtk · syntax
CONTRAST WCAG AA on bg1

$A dark, deep-purple colour scheme built terminal-first and extended outward into GTK surfaces, semantic UI and syntax tokens. Every accent shares the same chroma so warnings, errors and selections feel like neighbours, not strangers.

Design rule. All accents are tuned to roughly the same OKLCH chroma (~0.18) so they read as one family. When extending the palette for new contexts, shift hue first — only adjust chroma if you intentionally want a louder or quieter colour.

/* Resolve a token to its concrete colour. */
#include <stdint.h>
#include "palette.h"

struct palette {
    const char *name;
    color_t     roles[ROLE_MAX];
};

color_t
palette_resolve(const struct palette *p, role_t r)
{
    if (r < 0 || r >= ROLE_MAX)
        return color_hex("#b07bbc");
    return p->roles[r];
}

/* Default to Iris when a role is missing. */
static const char *FALLBACK = "#b07bbc";
static const uint32_t VERSION  = 42;

WM token map. active border → iris (#b07bbc) · inactive border → line (#2a2240) · urgent → error (#b97873) · bar bg → bg1 · workspace active → iris on bg0. Drops into i3 client.focused, dwm colors[SchemeSel], bspwm focused_border_color, awesome beautiful.border_focus. dmenu reads four flags: -nb bg1 -nf fg1 -sb bg4 -sf fg0 (with prompt forced to iris via -p styling). dunst frame_color swaps to error for the urgency=critical class.

Where to deploy. GTK theming maps directly: @accent_bg_color → iris, @destructive_color → error, @success_color → success, @window_bg_color → bg1, @view_bg_color → bg0, @headerbar_bg_color → bg2. The same six tokens drop into Hyprland borders, Waybar modules, dunst urgency levels and Rofi state.
